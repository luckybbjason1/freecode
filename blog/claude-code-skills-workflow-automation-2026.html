1. Claude Code Skills: 프로젝트별 컨텍스트 정의하기

Claude Code의 Skills 시스템은 .claude/skills/ 디렉토리에 SKILL.md 파일을存放하여, Claude Code에게 "이 프로젝트의 특화된 작업 방식을 알려주는" 방법입니다. 마치 프로젝트별 멘토를 만드는 것과 같습니다.

예를 들어, 한국 스타트업의 Next.js 프로젝트에서 다음과 같은 스킬을 정의할 수 있습니다:

# .claude/skills/nextjs-best-practices/SKILL.md
# 프로젝트: Next.js 14 + TypeScript + TailwindCSS
# 작성일: 2026-06-19

## 작업 흐름
1. 모든 페이지는 app/ 디렉토리에서 Server Components 우선
2. API 라우트는 app/api/ 에 배치, Zod 검증 필수
3. 스타일은 TailwindCSS 클래스, custom CSS 금지
4. 테스트는 Vitest + Testing Library

## 코드 스타일
- 함수형 컴포넌트만 사용 (Class Component 금지)
- async/await 패턴, Promise.then() 금지
- 한국어 주석 허용 (팀 내 표준)

이렇게 정의된 스킬은 Claude Code가 코드를 생성할 때 항상 참조됩니다. 2026년 5월 기준, LobeHub Skills Marketplace와 Claude Marketplace에서 500개 이상의 프리빌트 스킬을 다운로드할 수 있습니다.

2. Dynamic Workflows: 대규모 작업을 서브에이전트에 위임하기

Claude Code의 가장 강력한 기능 중 하나는 Dynamic Workflows입니다. 복잡한 작업을 여러 서브에이전트에 분할하여 병렬로 처리할 수 있습니다.

언제 Dynamic Workflows를 사용해야 하나요?

  • 코드베이스 감사: 전체 리포지토리의 보안 취약점, 코드 중복, 레거시 패턴 탐색
  • 대규모 마이그레이션: JavaScript → TypeScript, React → Next.js 등 프레임워크 전환
  • 교차 검증 연구: 여러 파일의 의존성을 분석하고 변경 영향도 평가

Dynamic Workflows 작동 방식

Claude Code가 워크플로우 스크립트를 작성하면, 사용자는 계획을 승인한 후 실행합니다. 각 서브에이전트는 독립된 세션(worktree)에서 실행되며, 최종 결과는 메인 세션에서 집계됩니다.

$ claude code --workflow migrate-to-ts
> JavaScript 파일 247개를 TypeScript로 마이그레이션해주세요.
> 각 파일별로 서브에이전트를 할당하고 병렬로 처리하세요.
> 마이그레이션 후 타입 검사(tsc --noEmit)를 실행하세요.

[Workflow 시작] 247개의 서브에이전트 생성 예정...
[진행률] ██████████ 100% (247/247 완료)
[결과] 234개 성공, 13개 수동 검토 필요

2026년 6월 기준, Claude Code의 Dynamic Workflows는 최대 100개 동시 서브에이전트를 지원합니다. 각 서브에이전트는 독립된 컨텍스트 윈도우를 가지므로, 메모리 누수나 컨텍스트 혼동을 걱정할 필요가 없습니다.

4. Hooks: Git 및 CI/CD 자동화

Claude Code의 Hooks 시스템은 Git 이벤트와 Claude Code 작업을 연결하여 워크플로우를 자동화합니다.

주요 Hook 유형

Hook트리거사용 사례
pre-commitgit commit 실행 전코드 스타일 검증, 보안 스캔
post-runClaude Code 세션 종료 후테스트 실행, 문서 업데이트
schedule정기적 스케줄주간 코드베이스 감사, 의존성 업데이트

예를 들어, pre-commit Hook에 Claude Code를 연결하면 커밋 전에 자동으로 코드 품질 검사를 실행할 수 있습니다.

# .claude/hooks/pre-commit.yaml
trigger: pre-commit
command: claude code --headless --prompt "이 커밋의 코드를 검토하고,
버그, 보안 취약점, 코드 중복이 있는지 확인하세요."
on_failure: reject_commit
timeout: 120s

5. Remote Control: CI 서버에서 Claude Code 실행하기

Claude Code의 Remote Control 기능을 사용하면 SSH가 연결된 서버에서 Headless 모드로 Claude Code를 실행할 수 있습니다. 이는 CI/CD 파이프라인과의 통합에 이상적입니다.

2026년 한국 개발자들 사이에서 인기 있는 CI/CD 통합 패턴:

# GitHub Actions 예시
name: Claude Code PR Review
on: pull_request

jobs:
review:
runs-on: ubuntu-latest
steps:
- uses: actions/checkout@v4
- name: Claude Code Review
run: |
claude code --headless \
--prompt "이 PR의 변경사항을 검토하고,
버그, 보안 취약점, 성능 문제를 찾아주세요."
env:
ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}

이 패턴을 사용하면 풀 리퀘스트가 열릴 때마다 Claude Code가 자동으로 코드 리뷰를 수행하고, 결과를 PR 댓글로 남깁니다. 한국 개발자 D님(풀스택 개발자, 5년 차)의 인터뷰: "GitHub Actions에 Claude Code를 연결한 후, 코드 리뷰 시간이 평균 30분에서 5분으로 줄었습니다."

6. Worktrees: 세션 격리로 병렬 작업하기

Claude Code의 Worktrees 기능은 각 세션을 독립된 Git 워크트리에서 실행합니다. 이는 Dynamic Workflows에서 각 서브에이전트가 서로 간섭 없이 작업할 수 있게 해줍니다.

예를 들어, 프론트엔드 리팩토링과 백엔드 API 개선을 동시에 진행할 때:

  • 서브에이전트 1: src/components/ 워크트리에서 프론트엔드 리팩토링
  • 서브에이전트 2: src/api/ 워크트리에서 백엔드 API 개선
  • 서브에이전트 3: tests/ 워크트리에서 테스트 수정

각 서브에이전트는 독립된 컨텍스트를 가지므로, 서로의 변경 사항을 간섭하지 않습니다. 최종 결과는 메인 세션에서 병합됩니다.

FAQ

Q1. Claude Code 스킬(Skills)이 무엇이며 어떻게 설치하나요?

A1. Claude Code 스킬은 .claude/skills/ 디렉토리에存放하는 재사용 가능한 작업 지침서입니다. SKILL.md 파일로 구성되며, 프로젝트 도메인별 컨텍스트와 워크플로우를 정의합니다. LobeHub Skills Marketplace(claudemarketplaces.com)에서 500개 이상의 프리빌트 스킬을 검색하고 설치할 수 있습니다.

Q2. 동적 워크플로우(Dynamic Workflows)는 무엇인가요?

A2. 동적 워크플로우는 Claude Code가 스크립트를 작성하여 여러 서브에이전트를 오케스트레이션하는 기능입니다. 코드베이스 감사, 대규모 마이그레이션, 교차 검증 연구 등에 사용됩니다. 각 서브에이전트는 독립된 세션에서 실행되며, 최종 결과는 메인 세션에서 집계됩니다.

Q3. Claude Code를 CI/CD 파이프라인에 통합하는 방법은?

A3. Claude Code는 --headless 모드로 실행되어 CI 파이프라인에서 비인터랙티브로 동작합니다. Hooks를 통해 pre-commit, post-run 이벤트에 연결할 수 있으며, Remote Control 기능으로 SSH 서버에서 원격 제어가 가능합니다. GitHub Actions에서 curl로 Claude Code API를 호출하는 패턴이 2026년 가장 인기 있는 통합 방식입니다.
